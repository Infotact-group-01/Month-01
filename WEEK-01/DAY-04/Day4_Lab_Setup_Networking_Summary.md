# Day 4 – Lab Setup & Networking Fundamentals

## Overview
During this session I set up the practical environment required for hands‑on cybersecurity work and revisited core networking concepts. The hands‑on lab reinforced the importance of a clean, reproducible environment and highlighted how networking underpins every security control.

## Key Topics Covered

1. **Virtual Machine Provisioning**
   - Created a fresh VM (Parrot Security) with 512 MB RAM and a 20 GB virtual disk.
   - Documented the steps for future reproducibility – snapshots, VM naming conventions, and resource allocation.

2. **Operating System & Tool Maintenance**
   - Ensured the OS was up‑to‑date and installed essential security tools (nmap, wireshark, metasploit).
   - Highlighted the habit of regularly applying patches to keep the lab secure.

3. **Networking Fundamentals Review**
   - Refreshed concepts of IP addressing, subnet masks, and basic routing.
   - Discussed how network segmentation reduces attack surface – directly relevant to the TIP’s firewall policies.

4. **Lab Documentation Practices**
   - Emphasized personal handwritten notes over relying solely on shared resources.
   - Noted that exam settings are often open‑book, encouraging the use of curated cheat‑sheets and quick‑reference guides.

5. **Course Context & Schedule**
   - Acknowledged the structured 45‑day training cadence (07:30 AM – 09:30 AM IST) which promotes disciplined learning.

## Goal of the Session
To establish a reliable, repeatable lab environment and solidify networking fundamentals so that I can safely test and validate the dynamic policy enforcer before deployment.
